# Hospital Management Database System

## Overview

This project is a relational database designed for managing hospital operations. The database stores information about patients, doctors, appointments, treatments, and departments while maintaining relationships between entities through primary and foreign keys.

The project was developed as part of a university database systems course.

## Features

- Patient management
- Doctor management
- Appointment scheduling
- Treatment records
- Department management
- Relational database design
- Data integrity through constraints

## Technologies Used

- SQL
- SQLite
- ER Diagram Modeling

## Database Structure

The database contains several tables, including:

- PATIENT
- DOCTOR
- APPOINTMENT
- TREATMENT
- DEPARTMENT

Relationships between tables are implemented using primary keys and foreign keys.
